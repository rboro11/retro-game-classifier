# Mario Game Identifier — Project Plan

## Goal
A multi-modal deep learning system that classifies **which Nintendo Mario game** a given
input (screenshot / video clip / audio clip) came from.  The architecture is designed to be
plug-and-play: swap in a new data directory and retrain without touching model code.

---

## Phase Roadmap

### Phase 1 — Binary Visual Classifier (Simplest Baseline)
**Goal:** Can a model tell SMB1 apart from SMB3?  
**Input:** Single screenshot (256×240 PNG)  
**Model:** Tiny custom CNN (3 conv layers)  
**Dataset:** Public only (no personal captures needed)  
**Data needed:**  
- SMB1 → `rafaelcp/smbdataset` — 737k frames, CC-BY-4.0 license ✅ PUBLIC  
- SMB3 → your NES Classic captures (personal) 🎮 YOU GATHER  
**Deliverable:** Working train/eval loop, >90% binary accuracy expected

---

### Phase 2 — Multi-Class Screenshot Classifier (3–5 Games)
**Goal:** Distinguish SMB1, SMB2, SMB3 from NES era  
**Input:** Single screenshot  
**Models:** Custom CNN, ResNet-18 (pretrained fine-tune), EfficientNet-B0  
**Data needed:**  
- SMB1 → rafaelcp/smbdataset ✅ PUBLIC  
- SMB2, SMB3 → your NES Classic captures 🎮 YOU GATHER  
- Optional: MobyGames API screenshots (requires free API key)  
**Deliverable:** Model comparison table, confusion matrix, per-class accuracy

---

### Phase 3 — Extended Visual Classifier (10–20 Games, Multi-Era)
**Goal:** NES + SNES + N64 + modern titles  
**Input:** Single screenshot  
**Models:** ResNet-50, EfficientNet-B2/B3, Vision Transformer (ViT-B/16)  
**Data needed:**  
- MobyGames API for SNES/N64/Wii/Switch era Mario games ✅ SEMI-PUBLIC  
  (requires MobyGames free API key: https://www.mobygames.com/info/api/)  
- Your own gameplay captures for games not on MobyGames 🎮 YOU GATHER  
- Super Mario Odyssey → smo-thumbnail-database (GitHub) ✅ PUBLIC  
**Deliverable:** Full multi-class benchmark across architectures

---

### Phase 4 — Audio Classifier
**Goal:** Identify game from a 5–15 second music/SFX clip  
**Input:** Audio spectrogram (mel-spectrogram image)  
**Model:** Small CNN on spectrograms, optionally CNN+LSTM  
**Data needed:**  
- NES titles → NES-MDB dataset (5,278 tracks, 397 NES games) ✅ PUBLIC  
  https://github.com/chrisdonahue/nesmdb  
- Non-NES titles → rip OST from your own game files 🎮 YOU GATHER  
  (use `ffmpeg` to extract audio from gameplay recordings)  
**Deliverable:** Standalone audio classifier, mel-spectrogram visualization

---

### Phase 5 — Video Classifier (Temporal)
**Goal:** Identify game from a short video clip (8–16 frames)  
**Input:** Frame stack or 3D tensor  
**Model:** CNN+LSTM, 3D-ResNet (slow path), TimeSformer  
**Data needed:**  
- SMB gameplay → taldatech/mario_gameplay HuggingFace (1280×720) ✅ PUBLIC  
- NES-VMDB — 98,940 gameplay clips, 389 NES games ✅ PUBLIC (arxiv 2404.04420)  
- Other titles → your own captures 🎮 YOU GATHER  
**Deliverable:** Temporal model, comparison vs single-frame model

---

### Phase 6 — Late Fusion Multi-Modal
**Goal:** Combine image + audio + (optionally) video into one prediction  
**Input:** Screenshot + audio clip  
**Model:** Separate backbones → concatenate embeddings → MLP head  
**Data needed:** All of the above  
**Deliverable:** Fused system, ablation study (image-only vs audio-only vs fused)

## Data Directory Convention (plug-and-play)

```
data/
  raw/
    SMB1/           ← drop PNGs or MP4s here
    SMB2/
    SMB3/
    SuperMarioWorld/
    SuperMario64/
    ...
  processed/
    frames/         ← auto-extracted by scripts/extract_frames.py
    spectrograms/   ← auto-generated by scripts/gen_spectrograms.py
    splits/         ← train/val/test CSVs, auto-generated
```

Adding a new game = create a new folder under `data/raw/<GameName>/`, drop your files,
run `scripts/build_dataset.py --mode all` — everything else is automatic.

---

## Recommended Capture Setup (for your raw data)
1. NES Classic HDMI → Elgato/AverMedia capture card → OBS → MP4
2. Record 5–10 minutes of varied gameplay per title (different worlds/levels)
3. `scripts/extract_frames.py` samples 1 frame/sec → ~300–600 images per session
4. Target: 500–2000 frames per game class for Phase 1–3
